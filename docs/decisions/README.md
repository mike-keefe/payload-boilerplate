# Architecture Decision Records (ADRs)

This folder records significant architectural decisions, including context,
options considered, and rationale.

Write an ADR when:

- You chose one technology or approach over alternatives
- A decision would be costly to reverse
- A future developer might question why something was done this way

## Template

```markdown
# ADR-[number]: [Short title]

**Date:** YYYY-MM-DD
**Status:** Proposed | Accepted | Deprecated | Superseded by ADR-[n]

### Context

What situation prompted this decision?

### Options considered

1. Option A — pros and cons
2. Option B — pros and cons

### Decision

What did we choose and why?

### Consequences

What becomes easier or harder as a result?
```
